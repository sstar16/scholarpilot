## 完成的工作

### 基础设施修复
- **创建 `.env` 文件**：发现根本原因——所有环境变量为空，服务无法连接数据库和Redis
- **nginx WebSocket 头**：`/api/` location 缺少 `Upgrade` / `Connection` 头，导致 WebSocket 静默失败
- **docker-compose volume 挂载**：新增 `./backend/app:/app/app` 挂载，开发改代码只需 `restart` 不需 `rebuild`

### 功能修复
| 问题 | 根因 | 修复 |
|------|------|------|
| Settings 页提供商名称空白 | 后端返回 `id` 字段，前端读 `provider_id` | `get_available_providers()` 改字段名，加 `model` 字段 |
| Settings 下拉菜单"无数据" | 删除 `PROVIDER_LABELS` 后 `el-option` 还在引用它 | 改为遍历 `providers` 数组 |
| Element Plus 对话框按钮乱码 | `{ locale: { name: 'zh-cn' } }` 写法错误 | `import zhCn` + `{ locale: zhCn }` |
| 检索结果 0 篇（中文查询） | `_extract_core_query` 提取原始中文发给英文数据库 | `build_query` 改 async，加 LLM 中英文翻译步骤 |
| 检索结果 0 篇（评分全0） | `expanded_terms = [base_query]` 把整句作为一个 term 匹配，永远不命中 | 拆分为单词列表 `base_terms = base_query.split()` |
| Worker 不使用用户配置的 LLM | LLM 配置只在 FastAPI 进程内存中，Celery worker 单独进程不知道 | 新增 `llm_config_store.py`，每次配置后存 Redis，worker 启动时加载 |
| 项目删除失败 | async SQLAlchemy ORM cascade 需预加载关联数据，直接 `db.delete()` 不触发 | 改为 `db.execute(delete(Project).where(...))` 触发 DB 级联 |
| 创建项目"502 Bad Gateway" | 容器重建后 nginx DNS 缓存了旧 IP | `docker-compose restart nginx` |
