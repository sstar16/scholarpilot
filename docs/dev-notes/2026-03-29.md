# Dev Notes — 2026-03-29

## 目标

跑通 MVP 端到端检索流程：创建项目 → Round 1 检索 → AI摘要 → 反馈提交 → Round 2 自动启动。

## 修复的 Bug

### 1. PubMed 摘要为空（已知，暂绕过）

**根因**：`eutils.ncbi.nlm.nih.gov` 在国内 Docker/WSL2 环境被 GFW 封锁，TLS 握手超时。
**修复**：`DISABLED_SOURCES=pubmed`，改用 EuropePMC（内容相同，有完整摘要，可从国内访问）。
**额外改进**：所有 fetcher 异常改为打印 `traceback.format_exc()`，便于后续排查。

### 2. 所有来源返回 0 篇文档

**根因**：LLM 把描述翻译成 6 个多词短语后直接 join，形成 22 词长查询字符串（如 `"tobacco heating system aerosol characterization nicotine delivery combustion-free"`）。OpenAlex/EuropePMC 对这类极精确查询返回 0 结果。
**修复**：`query_builder._get_english_query()` 将 LLM 返回的关键词短语展开为单词列表后取前 4 个；`_extract_core_query()` 也限制为前 4 词。
**教训**：学术 API 的全文搜索越短越好，3-4 词查询的召回率远高于 8+ 词。

### 3. SemanticScholar 持续 429

**根因**：SS 公开 API 无 API Key 时限速极严，并发请求下几乎必 429，指数退避也会消耗 30-50 秒。
**修复**：加了 4 次指数退避重试（2→4→8→30s，支持 `Retry-After` header）；同时将 `semantic_scholar` 加入 `DISABLED_SOURCES` 默认禁用，等到接入官方 API Key 后再启用。

### 4. arXiv 超时

**根因**：`export.arxiv.org` 在 Docker 容器内同样被封锁。
**修复**：`DISABLED_SOURCES` 加入 `arxiv,biorxiv,medrxiv`。

### 5. nginx 502（容器重建后）

**根因**：nginx 在启动时解析后端服务 IP 并缓存，`docker-compose up -d` 重建容器后 IP 变化但 nginx 不感知。
**修复**：每次 `up -d` 后都必须执行 `docker-compose restart nginx`。
**完整重建流程**：`docker-compose up -d backend worker && docker-compose restart nginx`

### 6. `positive_embedding` 列不存在

**根因**：ORM 模型中已定义 `Vector(384)` 列，但对应 Alembic 迁移文件不存在，且 `alembic/` 目录未挂载到容器 volume，手动创建的迁移文件也对容器不可见。
**修复步骤**：
1. 通过 `docker-compose exec postgres psql` 直接执行 `ALTER TABLE` 应急修复
2. `docker-compose.yml` 的 backend service 添加 `./backend/alembic:/app/alembic` volume mount
3. 创建正式迁移文件 `0002_add_embedding_cols.py`

**教训**：新增 ORM 列后必须同步创建迁移文件，且迁移目录要确认已挂载；`--autogenerate` 因缺少 mako 模板会报错，改为手动写迁移文件。

### 7. 反馈提交后前端不自动推进到下一轮

**根因**：`submitFeedback()` 没有为后端新建的 round 启动轮询；页面刷新后 `fetchRounds()` 也不恢复轮询。
**修复**：`stores/search.ts` 中：
- `fetchRounds()` 检查到 running/pending round 时自动调用 `startPolling()`
- `submitFeedback()` 完成后 re-fetch rounds 并对新 round 启动 polling

### 8. `proxy=None` 阻断代理

**根因**：httpx `AsyncClient` 中写了 `proxy=None`，显式禁用了系统代理，导致即使系统配置了 HTTP_PROXY 也无效。
**修复**：删除所有 fetcher 中的 `proxy=None` 参数，让 httpx 自动读取系统环境变量。

## 架构决策

### DISABLED_SOURCES 环境变量

在 `query_builder._select_sources()` 中通过 `os.getenv("DISABLED_SOURCES", "")` 读取逗号分隔的来源列表，在运行时过滤，无需改代码。`.env` 文件中设置：

```
DISABLED_SOURCES=pubmed,arxiv,biorxiv,medrxiv,semantic_scholar
```

### EuropePMC 作为 PubMed 替代

EuropePMC REST API 覆盖 PubMed 全部论文，且对中国大陆网络可访问。摘要字段在 `resultList.result[].abstractText` 中，已在 fetcher 中适配。

### 查询长度 4 词上限

测试结果：
- 3 词查询（"tobacco heat device"）→ OpenAlex 返回 6391 条
- 8 词查询（极精确）→ 返回 0 条

结论：学术 API 的查询越短召回越高，4 词是精确度和召回率的平衡点。
