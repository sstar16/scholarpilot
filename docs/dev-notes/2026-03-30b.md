# Dev Notes — 2026-03-30b

## 本次开发内容

### 数据源精细化控制（feat: 数据源精细化控制）

**背景**：用户反馈检索结果全为期刊论文，未见专利文献。排查发现专利功能已实现（LensPatentFetcher，LENS_API_TOKEN 已配置），但默认 `enable_patents: false`，且旧有设置界面只有两个开关（专利/临床试验），不够直观。

**变更点**：

#### 后端
- `backend/app/schemas/project.py`
  - 移除 `enable_patents: False` / `enable_clinical_trials: False`
  - 新增 `disabled_sources: []`（默认空列表 = 全部来源开启）
- `backend/app/services/query_builder.py` — `_select_sources()`
  - 专利（`lens_patent`）和临床试验（`clinical_trials`）不再需要显式开启，始终加入候选列表
  - 新增 `search_config.disabled_sources` 过滤，优先级高于旧字段
  - 向后兼容：旧项目若明确设置 `enable_patents: false` 且不含 `disabled_sources` 字段，仍按旧逻辑过滤

#### 前端
- `frontend/src/views/ProjectCreate.vue`
  - 数据源区由 2 个开关改为 9 个独立开关（3列网格），默认全开
  - 表单字段 `disabledSources[]` 替代 `enablePatents`/`enableClinicalTrials`
  - 提交时写入 `search_config.disabled_sources`
- `frontend/src/views/ProjectView.vue`
  - 新增右上角"搜索设置"按钮（本次迭代开始添加）
  - 弹窗升级为 9 个独立开关（2列网格），保存后合并到已有 search_config

**9 个可控数据源**：`openalex`、`europe_pmc`、`crossref`、`semantic_scholar`、`arxiv`、`biorxiv`、`medrxiv`、`lens_patent`、`clinical_trials`

**注意**：`DISABLED_SOURCES` 环境变量（全局）优先级高于项目设置，国内环境下 `arxiv/biorxiv/medrxiv/pubmed` 通常全局禁用。
