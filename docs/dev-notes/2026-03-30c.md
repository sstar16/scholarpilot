#### 3. Dockerfile 国内加速（重要坑）
原始 Dockerfile 使用 Debian 海外源，国内服务器 apt-get 极慢（50分钟未完成）。

**解决方案**：在 `backend/Dockerfile` 的 `RUN apt-get update` 前插入 sed 替换镜像源：
```dockerfile
RUN sed -i 's|deb.debian.org|mirrors.aliyun.com|g' /etc/apt/sources.list /etc/apt/sources.list.d/*.sources 2>/dev/null || true && \
    apt-get update && apt-get install -y \
    ...
```
pip 加清华源：
```dockerfile
RUN pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```
frontend npm 加 npmmirror：
```dockerfile
RUN npm install --registry=https://registry.npmmirror.com
```

**踩坑**：VS Code 粘贴长行会自动换行导致 Dockerfile 解析失败，必须手动逐行编辑，不能整段粘贴长命令。

### 更新版本流程（备忘）

```bash
# 1. 本地打 tag
git tag dev1 && git push origin dev1

# 2. 服务器拉取
cd /opt/scholarpilot/prod
git fetch --tags && git checkout dev1

# 3. 按改动类型操作
docker compose restart backend worker beat          # 只改了 Python
docker compose build frontend && docker compose up -d frontend && docker compose restart nginx  # 改了前端
docker compose build backend worker beat flower && docker compose up -d && docker compose restart nginx  # 改了依赖
```
