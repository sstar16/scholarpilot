# Dev Notes 2026-03-31

## 数据源调研 + 新数据源接入

### 问题背景

截图显示一次真实搜索结果：crossref: 5篇，europe_pmc: 1篇，其余全部为 0 篇。用户三个问题：
1. 各数据源有效性分析
2. 有无其他获取方式（调研 other-projects）
3. 中文优先但全是英文论文的原因

### 调研结论

**数据源失效原因：**
- `pubmed` → 中国网络封锁（TLS 超时）
- `openalex` → 应该能访问，0 篇是本次查询词翻译后不匹配（已加日志便于排查）
- `lens_patent` → 未配置 `LENS_API_TOKEN`（在 lens.org 免费申请）
- `semantic_scholar` → 被 DISABLED_SOURCES 禁用（频繁 429）
- `arxiv` → 中国网络封锁，被 DISABLED_SOURCES 禁用

**中文优先但全是英文的根本原因：**
- "chinese_first" scope 是空占位，中文数据源（万方/百度学术）尚未实现
- 查询词经 LLM 翻译为英文，所有现有数据源都是英文数据库

**other-projects 调研（5个项目）：**
- AutoResearchClaw：OpenAlex/S2/Google Scholar + circuit breaker 模式
- evil-read-arxiv：DBLP（CS 顶会）+ S2 联合搜索
- 无一实现中文学术或专利数据库

### 实施变更

#### 新增数据源

1. **DBLP** (`backend/app/services/fetchers/dblp.py`)
   - CS 顶会论文（CVPR/NeurIPS/ACL/ICML 等）
   - 免费 JSON API，无需鉴权
   - `https://dblp.org/search/publ/api`
   - cs/ai/computer 领域自动加入 sources

2. **百度学术** (`backend/app/services/fetchers/chinese.py`)
   - 中文论文主要来源
   - HTML 解析（非官方，接口可能变化）
   - 仅在「中文描述 + chinese_first scope」时触发
   - 使用原始中文查询词（不翻译）

#### 架构改动

- `QueryPlan` 新增 `original_chinese_query: Optional[str]` 字段
- `search_engine.py` 中文数据源使用 `original_chinese_query`，英文源用翻译后的 `base_query`
- `_select_sources()` 新增 `has_chinese_desc` 参数
- `OpenAlexFetcher` 加了 DEBUG 日志（查询词 + 命中数 + HTTP 状态）

#### 前端

- `ALL_SOURCES` 列表加入 DBLP 和百度学术
- 数据源标签（source stats）改为 `el-tooltip` 包裹，悬停显示：
  - 0篇：说明原因（如"需配置 LENS_API_TOKEN"、"国内访问受限"）
  - 有结果：显示成功条数
  - 错误：显示错误信息
